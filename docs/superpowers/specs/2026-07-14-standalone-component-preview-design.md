# 독립 컴포넌트 미리보기 설계

## 목적

Node.js, pnpm, 개발 서버 없이 디자이너가 구현 대상 7종의 형태와 상태를 브라우저에서 확인할 수 있는 단일 HTML 파일을 제공한다.

## 결과물

- 저장소 루트의 `figma-components-preview.html` 한 파일
- 파일을 더블클릭해 `file://` 환경에서 바로 실행
- 외부 스크립트, 패키지, 이미지, 네트워크 요청 없음
- CTA, CTA Small, CTA Insta, Text Field, Text Field Set, Survey Button, Indicator Bar를 상태별로 표시

## 화면 구성

- 상단에 “디자인 확인용 미리보기”와 실제 React 렌더링이 아니라는 안내를 표시한다.
- 컴포넌트마다 별도 카드와 상태명을 제공한다.
- 일반 컴포넌트의 검수 폭은 Figma와 동일한 350px, 공유 CTA는 171px을 사용한다.
- 모바일에서는 화면 너비를 넘지 않도록 축소되고, 넓은 화면에서는 카드 그리드로 배치한다.

## 시각 기준

- 현재 구현에 사용한 색상, 높이, 모서리 반경, 테두리, 타이포그래피 크기를 그대로 복제한다.
- 커스텀 폰트 파일은 외부 의존성이므로 포함하지 않고 시스템 한글 폰트 fallback을 사용한다.
- 링크 아이콘은 현재 컴포넌트와 동일한 SVG path를 HTML에 인라인으로 넣는다.

## 범위와 한계

- 디자인 확인만 위한 정적 복제 화면이다.
- React 컴포넌트 import, 실제 disabled 동작 검증, 접근성 검사, 빌드 검증을 대신하지 않는다.
- 실제 구현의 최종 검수는 실행 가능한 환경에서 `/playground`로 진행해야 한다.

## 확인 방법

1. `figma-components-preview.html`을 Finder에서 찾는다.
2. 파일을 더블클릭한다.
3. 브라우저에서 7종과 각 상태를 Figma 원본과 비교한다.
